# Master Landscape Catalogue

This document outlines the workflow for managing your studio's landscape vegetation data, with **`db/landscape/master_landscape.csv`** serving as the canonical database.

## 1. Core Principle: Local CSV as Source of Truth

All cleaned, standardized, and processed landscape data resides in `db/landscape/master_landscape.csv`. This CSV is optimized for fast, AI-assisted batch operations via Python scripts and quick visual lookups using Everything Search.

## 2. Data Sources & Ingestion

### 2.1 Google Sheet (Temporary Ingestion Point)
*   **Location:** Your "Landscape Schedule" Google Sheet (ID: `1kvXmUldDFyiXMJLQb2yPclFnHZOTsFHom7uGS7GGLPk`).
*   **Purpose:** This sheet is now considered a **temporary, messy ingestion point** for new or raw landscape data. It is *not* the master database.
*   **Sync to Local CSV:** To pull the latest data from the Google Sheet into your local CSV, run the following script:
    ```bash
    python tools/sync_gsheet_to_local_csv.py
    ```
    This script will read from the various sheets in your Google Sheet, clean and standardize the data, and **overwrite** `db/landscape/master_landscape.csv` with the fresh dataset.

### 2.2 Manual Data Append (Local CSV)
*   For smaller batches of new data or specific manual additions, you can use the `tools/append_landscape_data.py` script.
*   **Usage:** Prepare your data as a tab-separated string (including headers), save it to a temporary file, and pass the file path to the script:
    ```bash
    # Example: Create a temporary data file
    # echo "Botanical Name	Chinese Name
New Plant	新植物" > C:\Users
ealrendering\.gemini	mp
r-repo	emp_new_plant_data.txt

    # Run the append script
    python tools/append_landscape_data.py C:\Users
ealrendering\.gemini	mp
r-repo	emp_new_plant_data.txt
    ```
    This script will append the new data to `db/landscape/master_landscape.csv`, handling dynamic new columns and cleaning.

## 3. Data Structure (`db/landscape/master_landscape.csv`)

The `master_landscape.csv` file will maintain a consistent structure with the following columns (new columns will be added automatically as needed):

*   `Category`: (e.g., Tree, Shrub, Topiary, Groundcover)
*   `Botanical Name`
*   `Chinese Name`
*   `Height (mm)`
*   `Spread (mm)`
*   `3D Asset Brand`
*   `Model/Bundle`
*   `Status/Notes`
*   `Source Sheet`: (Original sheet name from Google Sheet)

## 4. AI-Assisted Data Operations (Python Scripts)

For any large-scale data cleaning, transformation, or batch modification (e.g., removing specific text from thousands of rows, re-categorizing based on conditions, deduplication), you will instruct me to **write or modify Python scripts** within the `tools/` directory. These scripts will operate directly on `db/landscape/master_landscape.csv`.

**Example Use Cases for AI-Generated Scripts:**
*   **Find & Replace at Scale:** "Write a script to find 'Nose' in the 'Notes' column and replace it with 'N' for all entries."
*   **Conditional Updates:** "Create a script to set the 'Category' to 'Large Tree' for all plants where 'Height (mm)' is greater than 5000."
*   **Data Combination/Merging:** "Develop a script to merge data from a new CSV into the master, prioritizing new information."

## 5. Visual Overview & Everything Search Integration

Your `db/landscape/master_landscape.csv` file is automatically enhanced for Everything Search with custom columns.

### 5.1 `.metadata.efu` File
*   A hidden file, `db/landscape/.metadata.efu`, is automatically generated by `tools/sync_gsheet_to_local_csv.py`.
*   This `.efu` file contains metadata for `master_landscape.csv` itself, defining custom columns that Everything Search can display.

### 5.2 Configuring Everything Search for Custom Columns
To leverage these custom columns in Everything Search:
1.  **Open Everything Search.**
2.  **Add Columns:** Right-click on any column header in the search results pane.
3.  Select **Add Columns...**.
4.  Scroll through the list and select the following custom columns (they will be listed by their `Custom Column Name`):
    *   `LandscapeCategory`
    *   `BotanicalName`
    *   `ChineseName`
    *   `HeightMM`
    *   `SpreadMM`
    *   `AssetBrand`
    *   `AssetModel`
    *   `StatusNotes`
    *   `SourceSheet`
5.  Click **OK**. These columns will now appear in your Everything Search results when you search within the `D:\GoogleDrive\RR_Repo\db\landscape` directory or directly for `master_landscape.csv`.

### 5.3 Searching with Custom Columns
Once configured, you can search and sort by these custom columns directly in Everything Search.
*   **Example Search:** To find all plants with a height of 500mm:
    `master_landscape.csv:HeightMM=500`
*   **Example Search:** To find all Trees from the VG sheet:
    `master_landscape.csv:LandscapeCategory=Tree master_landscape.csv:SourceSheet=VG`

## 6. Version Control

Since `db/landscape/master_landscape.csv` and `db/landscape/.metadata.efu` are part of your Git repository, all changes (whether via script or manual edits) are version-controlled, allowing you to track history, revert changes, and collaborate effectively.
