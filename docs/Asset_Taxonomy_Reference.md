# Asset Taxonomy & Tagging Reference

This document defines the master list of tags, categories, and brands used for 3D asset organization. 

## 1. Tagging Rules (The "Lean" Protocol)
*   **Minimalist:** Use as few tags as possible.
*   **Self-Defined:** Only use tags from this set.
*   **Non-Overlapping:** Tags must not have conceptual overlap.
*   **Singular:** Always use singular forms (e.g., `chair`, not `chairs`).
*   **Lowercase:** Tags are always `lower_case`.
*   **Single Words:** Prefer single-word tags (e.g., `diningchair` or `sports`).
*   **General Level:** Keep tags broad (e.g., `sports` instead of `volleyball`).
*   **No Obvious Tags:** Omit tags that are implied by the category.
*   **No Extensions:** Never use file extensions as tags.
